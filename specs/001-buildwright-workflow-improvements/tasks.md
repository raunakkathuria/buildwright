# Tasks: Buildwright Workflow Improvements

**Input**: Design documents from `/specs/001-buildwright-workflow-improvements/`

**Prerequisites**: plan.md, spec.md (required); research.md, data-model.md, contracts/, quickstart.md (loaded)

**Tests**: Per constitution Principle II, behaviour-changing work is test-first *where verifiable*. Flag removal (US6) was verified with `git grep` (zero references); a dedicated flag-absence gate was considered and **dropped as YAGNI** — a permanent script guarding one retired string isn't worth the maintenance. Prose-behaviour outcomes (stall recovery, faithful handoff, audit) are not unit-testable and are validated empirically via `quickstart.md`; those validation tasks are included explicitly and labelled.

**Organization**: Tasks grouped by user story. Ordering is dependency-driven (US6 + shared scaffolding first, since US4's "default" and other stories reference the autonomy model and native-capability map), then P1 fixes (US1/US2), then US3/US4/US5.

## Format: `[ID] [P?] [Story] Description`

- **[P]**: Can run in parallel (different files, no incomplete dependencies)
- All behaviour changes are edits to canonical `.buildwright/` (+ root `AGENTS.md`, constitution); generated `.claude/`/`.opencode/`/`.cursor/` dirs are NEVER hand-edited and are regenerated by `make sync`.

## Path Conventions

- Canonical source: `.buildwright/` at repo root; root docs `AGENTS.md`, `README.md`, `CHANGELOG`; governance `.specify/memory/constitution.md`; tooling `scripts/`, `Makefile`.
